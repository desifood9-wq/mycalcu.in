# mycalcu.in

A high-performance static calculator platform built for programmatic SEO at scale.

## Tech Stack

- Static HTML + CSS + Vanilla JavaScript
- Python build script (no framework required)
- GitHub Pages hosting
- GitHub Actions for automatic deployment

## Local Development

**Prerequisites:** Python 3.8+

```bash
# Clone the repo
git clone https://github.com/YOUR_USERNAME/mycalcu.in.git
cd mycalcu.in

# Build all calculator pages
python build.py

# Build a single calculator
python build.py --slug loan-emi-calculator

# Preview locally (Python built-in server)
python -m http.server 8000
# Open http://localhost:8000
```

## Adding a New Calculator

1. Add a new entry to `/data/calculators.json`
2. Add its content block to the `CONTENT` dict in `build.py`
3. Create `/assets/js/calculators/{slug}.js` with the calculator logic
4. Run `python build.py`
5. Push to `main` — GitHub Actions deploys automatically

## Deployment (GitHub Pages)

Deployment is fully automatic via GitHub Actions:
- Every push to `main` triggers a build + deploy
- Build runs `python build.py` which regenerates all pages and sitemap
- Site is live at `https://mycalcu.in` within ~2 minutes of pushing

**First-time setup:**
1. Push this repo to GitHub
2. Go to **Settings → Pages**
3. Set Source to **GitHub Actions**
4. Point your domain's DNS to GitHub Pages (see below)

## Architecture Rules (Do Not Break)

See `/docs/ARCHITECTURE.md` for full rules. Key constraints:

- Never modify URL structure (`/calculators/{slug}`)
- Never duplicate shared components
- Never introduce heavy frameworks
- Never add inline CSS or JS to HTML pages
- Always run `build.py` after editing templates or data
