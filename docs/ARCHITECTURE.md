# mycalcu.in — Architecture Reference

## Overview

mycalcu.in is a static calculator platform built for programmatic SEO (PSEO) scale.
The goal is to support 10,000+ calculator pages with minimal engineering effort per page.

## Tech Stack

- HTML (static, pre-rendered)
- CSS (single global stylesheet + per-page overrides if needed)
- Vanilla JavaScript (no frameworks)
- Python build script (pre-renders shared components into each page)

## URL Structure (PERMANENT — DO NOT CHANGE)

```
/calculators/{calculator-slug}
```

Examples:
- /calculators/salary-in-hand-calculator
- /calculators/fuel-cost-per-km-calculator
- /calculators/loan-emi-calculator

**Categories must never appear in the URL.**

## Shared Components Injection

Shared components are injected at build time by `build.py`.
The build script reads `calculator-template.html`, replaces
`<!-- INJECT:navbar -->`, `<!-- INJECT:footer -->`, `<!-- INJECT:header -->`
with the actual HTML content from `/shared/`, then writes the rendered
file to `/calculators/{slug}/index.html`.

This means:
- Zero runtime JS overhead for component loading
- Full HTML served to crawlers immediately
- Changing navbar.html + running build.py updates every page

## Calculator Page Structure (Required Sections)

Every calculator page must include these sections in order:

1. `<h1>` — Page title (calculator name)
2. Calculator input interface (form with labeled inputs)
3. Result display section
4. Explanation of the calculation method
5. Example calculation (with real numbers)
6. FAQ section (minimum 4 questions)
7. Related calculators (minimum 3 links)
8. Links to relevant guides

Content requirement: 500–800 words of explanatory text per page.

## SEO Requirements Per Page

Every page must include:
- Unique `<title>` tag
- `<meta name="description">` (unique per page)
- `<link rel="canonical">` pointing to canonical URL
- Open Graph tags (og:title, og:description, og:url, og:type)
- JSON-LD structured data (WebApplication schema)

## Performance Targets

- Page load: < 1 second
- No render-blocking JS
- Images: lazy loaded
- CSS: single minified file
- No external fonts loaded synchronously

## Adding a New Calculator (Process)

1. Add entry to `/data/calculators.json`
2. Run `python build.py`
3. The script generates `/calculators/{slug}/index.html`
4. Add URL to sitemap.xml (auto-generated by build script)
5. Deploy

No manual HTML editing required after the template is validated.
